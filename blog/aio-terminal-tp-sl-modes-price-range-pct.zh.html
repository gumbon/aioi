TP/SL 精通：AIO Terminal 中的价格、范围与 PCT 模式

AIO Indicator Team May 30, 2026 14 min read

每位活跃交易者都有自己对退出位置的心理模型。问题在于您的交易工具是否让这种模型的表达变得简单或困难。AIO Terminal 的三种 TP/SL 输入模式——价格（Price）、范围（Range）和 PCT——旨在匹配不同交易者实际思考退出的方式，而非强迫所有人采用单一的输入范式。

单一 TP/SL 输入模式的问题

Binance 的原生界面以绝对价格接受止损和止盈价位。如果您想在 ETH 交易入场价下方 $2 处设置止损，您需要先知道入场价，减去 $2，然后输入结果。如果您想要 1.5% 的止损，您需要计算当前价格的 1.5% 并从入场价中减去。每次入场价变化，您的计算也随之改变。

这在您思考风险的方式（以距离或百分比表示）与界面接受输入的方式（以绝对价格表示）之间强加了一层心理转换。在压力下，这层转换会导致输入错误。您输入了错误的数字，订单以错误的止损提交，您所承担的风险超过了计划。

AIO Terminal 通过同时提供全部三种表达格式消除了这层转换。您按照自己思考风险的方式设置——终端完成数学运算。

模式一：价格（绝对价位）

定义

价格（PRICE）模式以特定的绝对价格作为您的 TP 或 SL 输入。您输入"65000"作为 TP，订单将以 $65,000 的止盈触发价提交。无需计算——您输入的价格就是进入订单的价格。

适用场景

当您的退出价位直接来自图表分析时，PRICE 模式最为理想。如果您的图表显示 $65,000 处有重要阻力位，而您希望 TP 设在该价位略下方的 $64,950，您只需从图表上读取该价位并输入。无需中间步骤。图表分析与下单输入之间的关系是直接的。

同样，如果您在 $62,000 处定义了支撑位，并希望 SL 设在其下方的 $61,800，PRICE 模式可自然处理这种情况。您将退出锚定在市场结构上，而非入场价。

参考价格行为

在 PRICE 模式中，参考价格不用于计算——您直接提供最终价格。对于做多和做空选项卡（新建入场），PRICE 模式简单直接。对于 TP/SL 选项卡（管理现有持仓），参考价格是您当前的入场价，但仅作为参考显示，在 PRICE 模式下不用于计算。

典型用户

PRICE 模式适合使用结构化 TP/SL 定位的交易者：在需求区入场并在供应区设置 TP，在突破点入场并在下一个量度移动目标处设置 TP。图表告诉您确切的价位；PRICE 模式让您无需计算即可输入。

模式二：范围（固定距离）

定义

范围（RANGE）模式以距离值作为输入——即您希望 TP 或 SL 与参考价格相距的价格点数（或点差、美元数，取决于资产）。您输入"150"作为止损，AIO Terminal 将最终止损价计算为入场价减去 150（做多）或入场价加上 150（做空）。

方向自动处理。对于做多（LONG）仓位，AIO Terminal 知道止损在入场价下方（入场价 - 距离），TP 在入场价上方（入场价 + 距离）。对于做空（SHORT）仓位，逻辑反转。您无需考虑距离应用于哪个方向——系统会自动处理。

适用场景

当您的风险模型以美元距离或点位距离而非绝对价格表达止损和目标时，RANGE 模式最为理想。基于 ATR（平均真实范围）的止损非常适合此模式："我希望止损设在入场价下方 1.5 倍 ATR 处。"如果您交易品种的当前 ATR 为 200 点，您将 RANGE 止损设为 300。每笔交易都会自动获得正确距离的止损，无论入场价落在哪里。

对于在多笔交易中使用固定距离止损的短线交易者——"我的止损始终距入场 150 个 tick，目标始终为 300 个 tick"——RANGE 模式意味着您只需配置一次这些值，无需任何交易时计算即可应用于每笔交易。固定 R 倍数的一致性得以自动实现。

最终价格计算

对于在 $65,000 入场的做多，RANGE 止损设为 500：

  • SL 提交价：$65,000 - $500 = $64,500
  • 如果 RANGE TP 设为 1000：TP 提交价为 $65,000 + $1,000 = $66,000

对于在 $65,000 入场的做空，RANGE 止损设为 500：

  • SL 提交价：$65,000 + $500 = $65,500
  • 如果 RANGE TP 设为 1000：TP 提交价为 $65,000 - $1,000 = $64,000

对于市价入场（做多和做空选项卡），参考价格是提交时刻的标记价格（mark price）。对于限价入场，参考价格是您指定的限价。对于 TP/SL 选项卡（现有持仓），参考价格是您当前未平仓位的入场价。

典型用户

RANGE 模式适合基于 ATR 的风险交易者、固定 R 系统交易者，以及使用一致 tick 距离止损的短线交易者。任何风险模型以"距入场 X 点"而非"在该特定价位"描述退出的人，都会发现 RANGE 模式在整个交易日中节省了大量计算时间。

模式三：PCT（百分比）

定义

PCT 模式以百分比值作为输入。您输入"1.5"作为 TP，AIO Terminal 将 TP 价格计算为入场价 × (1 + 0.015)（做多），或入场价 × (1 - 0.015)（做空）。百分比应用于参考价格，得出的绝对价格作为订单提交。

适用场景

当您以百分比波动来思考风险时，PCT 模式最为理想——这也是大多数交易者用自然语言表达仓位风险的方式（"每笔交易我承担 1% 的风险"，"我的目标是 2% 的波动"）。PCT 模式无需手动将百分比转换为绝对价格，而是自动处理换算。

PCT 模式在比较不同资产的风险时也很有用。BTC 在 $65,000 时的 1.5% 止损与 ETH 在 $3,500 时的 1.5% 止损，其绝对距离各不相同。如果您希望跨交易保持一致的百分比风险，PCT 模式无论资产价格如何都能一致地应用。

最终价格计算

对于在 $65,000 入场的做多，PCT 止损设为 1.5%：

  • SL 提交价：$65,000 × (1 - 0.015) = $64,025
  • 如果 PCT TP 设为 2%：TP 提交价为 $65,000 × (1 + 0.02) = $66,300

对于在 $65,000 入场的做空，PCT 止损设为 1.5%：

  • SL 提交价：$65,000 × (1 + 0.015) = $65,975
  • 如果 PCT TP 设为 2%：TP 提交价为 $65,000 × (1 - 0.02) = $63,700

参考价格逻辑与 RANGE 模式相同：市价入场为标记价格，限价入场为限价，TP/SL 选项卡为当前入场价。

典型用户

PCT 模式适合基于百分比的风险交易者、根据每笔交易固定百分比风险调整仓位大小的交易者，以及任何希望跨不同资产和价格水平以标准化方式思考风险的人。对于以百分比衡量业绩的交易者而言，这也是最直观的模式。

混合模式：TP 与 SL 无需匹配

一个重要且常被忽视的功能：在同一笔交易中，您的 TP 和 SL 可以使用不同的模式。您可以将 SL 设为 RANGE 模式（因为止损基于 ATR），将 TP 设为 PRICE 模式（因为目标是特定阻力位）。或将 SL 设为 PCT（承担 1.5% 的风险），将 TP 设为 PRICE（目标 $66,500 的供应区）。

这种灵活性与成熟交易者实际构建交易的方式相匹配。止损定位逻辑通常与目标定位逻辑不同。止损可能是机械化的（入场价下方 2 倍 ATR），而目标是主观判断的（图表上下一个重要供应区）。AIO Terminal 不会强迫这两种不同的分析方法采用相同的输入模式。

TP/SL 选项卡：管理未平仓位

除了做多和做空下单选项卡，AIO Terminal 还包含一个专用的 TP/SL 选项卡，用于修改已开仓位的退出点。该选项卡以您的实际入场价作为所有三种模式的参考点。

实际应用案例：您在 $65,000 做多入场，交易已移动至 $65,800，您希望将止损收紧至保本价加 50 点。首先通过取消（Cancel）选项卡取消现有 SL，然后前往 TP/SL 选项卡，选择 RANGE 模式，输入"50"作为新的 SL 距离。AIO Terminal 计算 $65,000 + $50 = $65,050，并将新止损提交至 Binance。您无需输入绝对止损价——您将其表达为"入场价上方 50 点"，系统处理了数学运算。

更新 TP/SL 非常简单：只需随时在 TP/SL 选项卡中重新设置即可。AIO Terminal 会在提交新订单之前自动取消该仓位的所有现有 TP/SL 订单——无需手动取消步骤。

PH/PL 快速填充适用于全部三种模式

做多、做空和 TP/SL 选项卡均包含 PH（枢轴高点）和 PL（枢轴低点）快速填充按钮。其行为会根据您选择的输入模式自动适配：

  • 价格模式（Price mode）：点击 PL 将以绝对枢轴低点价格填充 SL 字段。点击 PH 将以绝对枢轴高点价格填充 TP。直接填充，无需转换。
  • 范围模式（Range mode）：点击 PL 计算 |pivot_low − entry| 并以该距离填充范围字段。您无需计算枢轴距入场有多远——终端会自动完成。
  • PCT 模式（PCT mode）：点击 PL 计算 |pivot_low − entry| / entry × 100 并填充百分比字段。当您以百分比风险方式思考，但希望 SL 锚定于实际市场结构时非常有用。

这意味着无论您的风险管理系统使用何种单位——绝对价格、点位距离还是百分比——都可以使用基于枢轴的止损和目标，无需手动在它们之间转换。从枢轴价格到范围或 PCT 的换算，在您点击 PH 或 PL 的瞬间自动完成。

选择您的默认模式

对于大多数交易者，一种模式将主导其工作流程并成为默认选择。以下是一个简单的决策框架：

  • 使用 PRICE 模式如果：您的交易设置主要来自图表结构（供应/需求区域、支撑/阻力位、量度移动）
  • 使用 RANGE 模式如果：您的风险模型在交易中始终使用基于 ATR 的止损或固定 tick 距离规则
  • 使用 PCT 模式如果：您根据每笔交易的固定百分比风险调整仓位大小，或希望跨不同资产保持一致的百分比风险

您的 TP 和 SL 可以根据各自的确定方式使用不同的默认值。大多数结构化交易者对 TP 使用 PRICE 模式（目标特定区域），对 SL 使用 RANGE 或 PCT 模式（固定机械化风险）。两者同样有效。这种灵活性的存在，正是因为表达风险没有唯一"正确"的方式——只有与您的分析相匹配、并帮助您在压力下避免输入错误的方式。
