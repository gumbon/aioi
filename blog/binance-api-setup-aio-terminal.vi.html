Hướng Dẫn Tạo API Key Binance cho AIO Terminal

AIO Indicator Team May 31, 2026 8 min read

Trước khi AIO Terminal có thể thực hiện lệnh giao dịch thay bạn, bạn cần kết nối nó với tài khoản Binance thông qua API key. Hướng dẫn này đi qua toàn bộ quá trình thiết lập: tạo API key, bật các quyền cần thiết, bảo mật bằng giới hạn IP và đăng nhập vào AIO Terminal lần đầu tiên.

Trước khi bắt đầu

AIO Terminal dành cho tất cả thành viên gói VIP. Sau khi đăng ký, hãy liên hệ chúng tôi qua Telegram (@nguyenthl) để nhận thông tin đăng nhập AIO Terminal và các địa chỉ IP tin cậy cần dùng ở bước 5 bên dưới.

Bước 1: Mở API Management trên Binance

Đăng nhập vào tài khoản Binance tại binance.com. Nhấp vào biểu tượng hồ sơ ở góc trên bên phải để mở menu tài khoản. Trong menu thả xuống, nhấp vào API Management.

Bạn sẽ đến trang API Management, nơi liệt kê tất cả các API key hiện có trên tài khoản. Nếu đây là lần đầu tiên, danh sách sẽ trống.

Mẹo

Đặt nhãn rõ ràng cho API key, ví dụ "AIO Terminal", để dễ nhận biết sau này. Nếu bạn cần thu hồi hoặc kiểm tra, nhãn mô tả sẽ giúp tiết kiệm thời gian.

Bước 2: Tạo API Key Mới

Nhấp vào nút Create API. Binance sẽ yêu cầu bạn chọn loại key:

  • System generated — Binance tự tạo key và secret cho bạn. Đây là lựa chọn đúng cho AIO Terminal.
  • Self-generated (Ed25519) — dành cho các trường hợp nâng cao yêu cầu tạo key tùy chỉnh. Không cần dùng ở đây.

Chọn System generated, sau đó nhập nhãn (ví dụ: "AIO Terminal") và nhấp Next. Binance sẽ yêu cầu bạn hoàn tất bước xác minh danh tính — thường bao gồm xác nhận email, SMS OTP hoặc Google Authenticator tùy theo cài đặt 2FA của tài khoản. Hoàn tất xác minh để tiếp tục.

Bước 3: Lưu API Key và Secret Key — Làm Ngay Bây Giờ

Sau khi xác minh, Binance hiển thị API Key và Secret Key mới của bạn. Đây là lần duy nhất Secret Key được hiển thị. Khi bạn đóng trang này, Secret Key sẽ biến mất vĩnh viễn khỏi giao diện Binance — không có nút "hiển thị secret" nào sau đó.

Quan trọng

Sao chép cả API Key và Secret Key rồi lưu ở nơi an toàn (trình quản lý mật khẩu là lý tưởng nhất) trước khi rời trang. Nếu mất Secret Key, bạn phải xóa API key này và tạo lại từ đầu.

Hai giá trị này là thứ bạn sẽ nhập vào cài đặt của AIO Terminal để kết nối tài khoản Binance:

  • API Key — một chuỗi ký tự và số dài, trông giống UUID. Có thể chia sẻ an toàn với AIO Terminal.
  • Secret Key — dài tương đương, ngẫu nhiên tương đương. Hãy xem nó như mật khẩu. AIO Terminal mã hóa và lưu trữ nó phía máy chủ; không bao giờ hiển thị dưới dạng văn bản thuần.

Bước 4: Bật Quyền Futures

Sau khi API key được tạo, bạn sẽ đến trang cài đặt của nó (hoặc quay lại API Management và nhấp vào key). Bạn sẽ thấy danh sách các hộp kiểm quyền. Mặc định, chỉ có Read Info được bật.

Để cho phép AIO Terminal đặt và quản lý lệnh, bạn phải bật:

  • Enable Futures — bắt buộc để AIO Terminal gửi lệnh Long/Short, đặt TP/SL, đặt trailing stop và đóng vị thế trên tài khoản Binance USDⓈ-M Futures của bạn.
KHÔNG bật quyền rút tiền

Tuyệt đối không bật "Enable Withdrawals" trên API key dùng với bất kỳ công cụ bên thứ ba nào. AIO Terminal không yêu cầu quyền rút tiền và sẽ không bao giờ đề nghị điều đó. Giữ quyền rút tiền bị tắt có nghĩa là AIO Terminal — và bất kỳ ai có thể xâm phạm API key — không thể chuyển tiền ra khỏi tài khoản của bạn. Đây là ranh giới bảo mật quan trọng.

Sau khi chọn Enable Futures, lưu các quyền. Binance có thể yêu cầu xác nhận 2FA thêm một lần nữa.

Bước 5: Giới Hạn Truy Cập Chỉ Từ IP Tin Cậy

Đây là cài đặt bảo mật quan trọng nhất. Trên cùng trang cài đặt API key, tìm tùy chọn có nhãn "Restrict access to trusted IPs only" và bật nó lên.

Khi bật tùy chọn này, API key của bạn chỉ có thể được sử dụng từ các địa chỉ IP cụ thể mà bạn liệt kê. Ngay cả khi API Key và Secret Key bị đánh cắp, chúng sẽ vô dụng với kẻ tấn công gọi từ bất kỳ IP nào khác — máy chủ Binance từ chối mọi yêu cầu không xuất phát từ IP được phép của bạn.

Nhập Địa Chỉ IP Nào

Đội ngũ AIO Indicator sẽ cung cấp cho bạn các địa chỉ IP tin cậy cần thêm vào whitelist khi bạn nhận thông tin đăng nhập AIO Terminal. Đây là các địa chỉ IP của máy chủ AIO Terminal — máy duy nhất thực hiện các lệnh gọi API thay bạn.

Nhận địa chỉ IP của bạn

Khi bạn đăng ký gói VIP và liên hệ chúng tôi qua Telegram, chúng tôi sẽ gửi cho bạn:
• Thông tin đăng nhập AIO Terminal (tên đăng nhập + mật khẩu ban đầu)
• Các địa chỉ IP tin cậy cần thêm vào whitelist trong cài đặt Binance API

Nhập các IP này vào trường "Restrict access to trusted IPs only" trên Binance, phân cách bằng dấu phẩy hoặc nhập từng dòng tùy theo giao diện hiện tại của Binance.

Sau khi nhập IP và lưu, API key của bạn đã được cấu hình hoàn chỉnh. Sự kết hợp "Futures bật, rút tiền tắt, giới hạn IP" là tư thế bảo mật chuẩn vàng cho API key của công cụ giao dịch.

Bước 6: Đăng Nhập vào AIO Terminal

Mở terminal.aioindicator.com trên bất kỳ trình duyệt hiện đại nào. Bạn không cần cài đặt gì — AIO Terminal hoàn toàn dựa trên web và hoạt động trên máy tính để bàn, máy tính bảng và điện thoại di động.

Đăng nhập bằng tên đăng nhập và mật khẩu ban đầu được cung cấp bởi đội ngũ AIO Indicator. Lần đăng nhập đầu tiên, bạn sẽ được yêu cầu đặt mật khẩu mới. Chọn mật khẩu mạnh, độc đáo và lưu vào trình quản lý mật khẩu.

Kết Nối Tài Khoản Binance

Sau khi đăng nhập, điều hướng đến Settings trong AIO Terminal. Nhập:

  • API Key — chuỗi key từ Bước 3
  • Secret Key — chuỗi secret từ Bước 3

AIO Terminal sẽ xác thực kết nối bằng cách thực hiện một lệnh gọi đọc thử đến tài khoản Binance của bạn. Nếu kết nối thành công, các vị thế và lệnh đang mở của bạn sẽ xuất hiện trong bảng điều khiển trong chu kỳ đồng bộ dữ liệu tiếp theo (mặc định: 5 giây).

Cách AIO Terminal lưu trữ thông tin xác thực của bạn

API Key và Secret Key của bạn được mã hóa trên máy chủ AIO Terminal. Chúng không bao giờ được lưu dưới dạng văn bản thuần, không bao giờ được truyền không mã hóa và không bao giờ hiển thị cho đội ngũ AIO Indicator. AIO Terminal chỉ sử dụng API key Binance của bạn — không bao giờ dùng mật khẩu tài khoản Binance.

Thêm Tài Khoản Binance Thứ Hai (Tùy Chọn)

AIO Terminal hỗ trợ tối đa 2 tài khoản Binance cùng lúc. Để thêm tài khoản thứ hai, lặp lại quy trình tạo API key ở trên trên tài khoản Binance thứ hai của bạn, sau đó nhập thông tin xác thực đó vào Settings của AIO Terminal ở ô tài khoản thứ hai.

Bạn có thể chuyển đổi giữa các tài khoản trong giao diện Terminal. Điều này hữu ích cho các trader vận hành các tài khoản riêng biệt theo các chiến lược khác nhau — ví dụ: một tài khoản cho vị thế futures theo xu hướng và tài khoản kia cho scalping ngắn hạn.

Thử Nghiệm Trên Binance Testnet Trước

Nếu bạn mới bắt đầu giao dịch futures hoặc mới dùng AIO Terminal, chúng tôi khuyến khích mạnh mẽ việc kết nối với Binance Testnet trước khi vào thực chiến. Testnet cung cấp môi trường futures mô phỏng với tiền ảo — tất cả tính năng AIO Terminal đều hoạt động giống hệt: Long, Short, TP/SL, Trailing Stop, Close — nhưng không có tiền thật nào bị rủi ro.

Để sử dụng Testnet:

  1. Tạo tài khoản tại testnet.binancefuture.com (tách biệt với tài khoản Binance chính của bạn)
  2. Tạo API key trên trang testnet theo cùng quy trình mô tả ở trên
  3. Trong Settings của AIO Terminal, chuyển sang chế độ Testnet và nhập thông tin xác thực API testnet của bạn
  4. Luyện tập cho đến khi thành thạo, sau đó chuyển lại mainnet với thông tin xác thực thực của bạn

Xử Lý Sự Cố Thường Gặp Khi Cài Đặt

Kết nối thất bại: "Invalid API key"

Kiểm tra lại xem bạn đã sao chép đúng toàn bộ chuỗi API Key chưa, không có khoảng trắng thừa. API key phân biệt chữ hoa/thường và thường dài 64 ký tự. Nếu bạn sao chép đúng từ Binance nhưng vẫn thất bại, nguyên nhân phổ biến nhất là quyền Futures chưa được lưu — quay lại Binance API Management và xác nhận "Enable Futures" đã được chọn và lưu.

Lệnh gửi nhưng bị từ chối: "IP address not whitelisted"

Lỗi này từ Binance nghĩa là bạn đã bật giới hạn IP nhưng IP của máy chủ AIO Terminal không nằm trong whitelist, hoặc bạn đã nhập IP sai. Liên hệ chúng tôi qua Telegram và chúng tôi sẽ xác nhận lại các địa chỉ IP hiện tại. Lưu ý rằng IP máy chủ AIO Terminal không thay đổi thường xuyên, nhưng nếu đội ngũ cập nhật cơ sở hạ tầng, bạn sẽ được thông báo và yêu cầu cập nhật whitelist.

Không thể bật Futures: "Complete futures account setup first"

Binance Futures yêu cầu kích hoạt tài khoản futures trước khi có thể cấp quyền futures ở cấp API. Đăng nhập vào Binance, điều hướng đến mục Futures và hoàn tất quá trình onboarding futures (thường bao gồm đọc bản công bố rủi ro và xác nhận kinh nghiệm giao dịch của bạn). Sau khi tài khoản futures được kích hoạt, quay lại API Management và bật quyền Futures trên key của bạn.

Mất Secret Key sau khi đóng trang

Secret Key không thể lấy lại sau khi màn hình tạo bị đóng. Bạn phải xóa API key bị mất/xâm phạm khỏi Binance API Management và tạo lại từ đầu. Key cũ cần được xóa ngay lập tức để tránh rủi ro bảo mật từ một key chưa được ghi lại còn tồn tại ở phía Binance.

Danh Sách Kiểm Tra Nhanh

Dùng danh sách kiểm tra này để xác nhận quá trình cài đặt của bạn đã hoàn chỉnh và bảo mật:

  • ☐ API key Binance đã được tạo với nhãn dễ nhận biết (ví dụ: "AIO Terminal")
  • ☐ API Key và Secret Key đã được lưu trong trình quản lý mật khẩu
  • ☐ Quyền Enable Futures đã được chọn và lưu
  • ☐ Enable Withdrawals — CHƯA chọn (nên để tắt)
  • ☐ "Restrict access to trusted IPs only" đã bật với các IP AIO Terminal đã nhập
  • ☐ Đã đăng nhập vào terminal.aioindicator.com bằng thông tin xác thực được cung cấp
  • ☐ Mật khẩu ban đầu đã được đổi thành mật khẩu mạnh, độc đáo
  • ☐ Thông tin xác thực API đã nhập trong Settings của AIO Terminal — kết nối đã xác nhận

Khi tất cả các mục đã được kiểm tra, AIO Terminal của bạn đã hoạt động đầy đủ. Mở vị thế đầu tiên bằng tab Long hoặc Short và trải nghiệm sự khác biệt về tốc độ so với giao diện gốc của Binance.

Nếu bạn có bất kỳ câu hỏi nào trong quá trình cài đặt, hãy liên hệ chúng tôi qua Telegram tại @nguyenthl — chúng tôi sẽ hướng dẫn bạn từng bước.
