AIO Terminal 고급 기능: 모든 모드의 PH/PL, 키보드 단축키, 멀티 계정 & 더 보기

AIO Indicator Team May 31, 2026 10 min read

대부분의 트레이더는 1~2번의 클릭 진입, TP/SL, 청산 같은 주요 기능을 통해 AIO Terminal을 접합니다. 하지만 숙련된 사용자와 초보자를 구분하는 것은 세부 기능입니다 — 모든 입력 모드에서 작동하는 PH/PL 피벗 단축키, 즉각적인 탭 이동을 위한 키보드 단축키, "Last" 버튼, 개별 포지션에 대해 행동할 수 있는 기능. 이 가이드에서 모두 다룹니다.

1. PH 및 PL 버튼: 클릭 한 번으로 구조적으로 유효한 손절 설정

AIO Terminal의 SL 입력 필드 옆에는 두 개의 작은 버튼이 있습니다: PH(피벗 고점)와 PL(피벗 저점). 이 버튼들은 선택된 심볼과 타임프레임의 가장 최근 유의미한 스윙 고점 또는 스윙 저점으로 SL 필드를 채우는 원클릭 단축키입니다.

기능 설명

PH를 클릭하면 AIO Terminal이 가장 최근 피벗 고점 — 현재 가격 이전에 형성된 스윙 고점 — 을 불러옵니다. PL을 클릭하면 가장 최근 피벗 저점을 불러옵니다. 해당 가격 값이 SL 필드에 직접 입력됩니다.

롱 트레이드의 경우: 일반적으로 PL을 클릭하여 가장 최근 스윙 저점 아래에 손절을 설정합니다. 가격이 그 수준 아래로 돌아오면 롱 셋업이 무효화되며 — 이것이 바로 구조적으로 유효한 손절이 위치해야 할 곳입니다. 숏 트레이드의 경우: PH가 가장 최근 스윙 고점을 제공하며, 이는 숏의 구조적 무효화 수준입니다.

이것이 중요한 이유

손절 배치의 고전적인 실수는 시장 구조를 참고하지 않고 라운드 넘버나 임의의 고정 거리에 손절을 설정하는 것입니다. "진입가 아래 1.5%"의 손절은 시장이 실제로 거래를 무효화하는 지점과 완전히 무관할 수 있습니다. 가장 최근 스윙 저점 바로 아래의 손절은 시장 구조를 인식한 배치입니다 — 돌파될 경우 거래가 잘못되었음을 실질적으로 알려주는 수준에 위치합니다.

그 수준을 수동으로 찾으려면: 차트로 전환하고, 가장 최근 유의미한 피벗을 육안으로 식별하고, 가격 수준을 읽고, AIO Terminal로 다시 전환하고, 가격을 입력해야 합니다. 시간 압박 하에서 이 과정은 지연과 전사 오류를 모두 유발합니다. PH/PL은 이를 버튼 한 번 누르는 것으로 압축합니다 — 그리고 피벗을 어떤 타임프레임에서 가져올지 선택할 수 있으므로, 스캘퍼는 5분 피벗을 사용하고 스윙 트레이더는 4시간 피벗을 사용할 수 있습니다.

PH/PL은 세 가지 입력 모드 모두에서 작동

PH와 PL 버튼은 PRICE 모드에만 국한되지 않습니다 — Range와 % 모드에서도 자동 변환과 함께 작동합니다. Range 모드에서 PL을 클릭하면 AIO Terminal이 |피벗_가격 − 진입_가격|을 계산하여 Range 필드에 그 거리를 채웁니다. % 모드에서는 |피벗_가격 − 진입_가격| / 진입_가격 × 100을 계산하여 퍼센트를 채웁니다. 이 수학을 직접 계산할 필요가 없습니다.

즉, 포지션 사이징 목적으로 Range 또는 % 단위로 리스크를 설정하면서도, 클릭 한 번으로 실제 구조적 피벗에서 실제 수준을 채울 수 있습니다. 워크플로우는: 입력 모드 선택 → PH 또는 PL 클릭 → 터미널이 해당 모드에 맞는 올바른 값을 채움 → 필요시 약간 조정 → 확인.

2. "Last" 버튼: 수동 가격 입력 없이 시장 가격으로 지정가 주문

Long 및 Short 주문 탭의 Limit Price 필드 옆에 Last라고 표시된 버튼이 있습니다. 이 버튼을 누르면 선택된 심볼의 현재 마지막 체결 가격이 지정가 필드에 채워집니다 — 거의 제로 레이턴시의 실시간 WebSocket 피드에서 가져옵니다.

가능해지는 워크플로우

Last 버튼의 가장 일반적인 사용은 현재 시장 가격에 지정가 주문을 입력하여 거의 100% 체결률로 메이커 수수료 체결을 얻는 것입니다. 순서:

  1. 차트에서 진입 신호 확인
  2. AIO Terminal로 전환 (Long 또는 Short 탭이 이미 해당 심볼 표시 중)
  3. Last 탭 — 현재 시장 가격이 지정가 필드에 채워짐
  4. Long 또는 Short 탭 — 미리 설정된 TP와 SL과 함께 해당 가격에 지정가 주문 제출

차트 신호에서 주문 제출까지 총 시간: 2~4초. 주문은 시장 가격에 지정가 주문으로 접수됩니다 — 스프레드를 넘는 시장가 주문이 아닙니다. 활발한 페어에서는 테이커 수수료(0.05%) 대신 메이커 수수료(0.02%)로 밀리초 내에 체결됩니다.

시장가 주문을 사용하지 않는 이유

수수료 계산은 단순하지만 과소평가하기 쉽습니다. $10,000 포지션의 경우: 시장가 수수료 $4 vs 지정가 수수료 $2. 하루 10거래: $40 vs $20. 20 거래일: $800 vs $400. 연간: $9,600 vs $4,800 — 전략의 단 하나도 바꾸지 않고 $4,800 차이. Last 버튼은 사용하는 모든 진입에서 이 절약을 자동으로 만들어 줍니다.

지정가 주문의 체결 우려 — "체결 안 되면 어떡하지?" — 는 마지막 가격에는 적용되지 않습니다. 마지막 가격은 시장이 방금 거래된 곳입니다. 그 가격에 지정가 매수를 하는 것은 불과 밀리초 전에 매수자와 매도자가 활동하던 수준의 대기열에 합류하는 것입니다. 유동성 있는 시장에서는 즉시 체결됩니다. Last 버튼은 지정가 주문 수수료로 시장가 주문 속도를 제공합니다.

지정가 vs 시장가: 둘 다 원터치

AIO Terminal은 어느 주문 유형도 강제하지 않습니다. 시장가 주문도 동등하게 접근 가능합니다 — Limit 버튼 대신 Market 버튼 한 번. 체결 확실성이 절대적으로 필요한 진입(불리한 가격의 부분 체결이 허용되는 돌파)의 경우, 시장가 주문이 올바른 선택입니다. 신호가 반초의 여유를 허용하는 대부분의 진입에서는 Last + 지정가가 최적의 실행 경로입니다.

3. 특정 포지션에 대한 행동: "전체 또는 없음"만이 아닌

AIO Terminal에서 가장 흔히 오해받는 측면 중 하나는 "청산" 및 "취소" 기능이 유일한 옵션이 아니라는 것입니다. Close, Cancel, TP/SL을 포함한 모든 액션 탭은 여러 포지션이 열려 있을 때 목록에서 특정 포지션을 선택할 수 있습니다.

포지션 선택기

여러 개의 열린 포지션이 있을 때 (예: BTC 롱과 ETH 숏이 동시에 실행 중), 액션 탭에 포지션 선택기가 표시됩니다. 실행 전에 행동할 포지션을 선택합니다. 행동 — 청산, 주문 취소, 새 TP/SL 설정, 트레일링 스탑 활성화 — 은 선택된 포지션에만 적용됩니다. 나머지 포지션은 그대로 유지됩니다.

포지션 선택 vs "전체" 버튼 사용 시점

포지션 선택을 사용할 때:

  • 다른 포지션을 유지하면서 한 포지션의 TP/SL을 청산하거나 업데이트할 때
  • 다른 포지션의 SL에 영향을 주지 않고 한 포지션의 대기 중 TP 주문을 취소할 때 (타겟을 변경했기 때문에)
  • 시장 상황 변화 후 특정 포지션의 TP/SL을 새 수준으로 재설정할 때

"전체" 버튼을 사용할 때:

  • 뉴스가 터져 즉시 모든 것을 평탄화해야 할 때 (청산)
  • 선택된 심볼의 모든 미결 주문을 한 번에 취소할 때 (취소)
  • 세션을 마치고 상태를 초기화할 때 (취소 + 청산 순서로)

개별 포지션을 타겟팅할 수 있는 기능이 AIO Terminal을 단순한 빠른 주문 진입 인터페이스가 아닌 완전한 포지션 관리 도구로 만드는 것입니다. 실행 중인 트레이드 중에 하는 정밀한 조정과 상황이 빠르게 움직일 때 필요한 긴급 청산 모두를 처리합니다.

4. 키보드 단축키: 즉각적인 탭 이동

AIO Terminal의 모든 탭에는 전용 키보드 단축키가 있습니다. 탭 버튼을 클릭하는 대신, 해당 키를 누르면 즉시 이동합니다. 이것은 트레이드 실행 시간을 줄이는 가장 레버리지 효율이 높은 습관 중 하나입니다.

기본 단축키 매핑

키 탭
1Long 주문
2Short 주문
4포지션 청산
6TP/SL
7주문 취소
8Analytics

설정에서 단축키 재매핑

모든 단축키는 Terminal 설정에서 재매핑할 수 있습니다. 워크플로우가 TP/SL과 Analytics에 집중되어 있다면, 빠른 접근을 위해 그것들을 키 1과 2로 재매핑할 수 있습니다. 주로 진입을 관리한다면, 기본값을 유지하세요. 매핑은 세션 간에 유지됩니다 — 한 번 설정하면 매번 사용됩니다.

근육 기억 구축

키보드 단축키의 가치는 친숙함과 함께 복리로 증가합니다. 같은 키 매핑을 일주일 동안 사용한 후에는 탭 간 이동이 반사적으로 됩니다 — 인지적 부담 없이, 탭을 찾기 위해 화면을 볼 필요도 없습니다. "Last" 버튼과 PH/PL과 결합하면, 차트 신호에서 라이브 주문까지의 전체 진입 워크플로우가 클릭 없이 3초 이내에 완료됩니다.

5. 두 개의 Binance 계정 동시 트레이딩

AIO Terminal은 동시에 최대 두 개의 Binance 계정을 연결하는 것을 지원합니다. 각 계정은 자체 API 키를 통해 연결되고 터미널 내에서 독립적으로 작동합니다.

이중 계정 사용 사례

트레이더들은 여러 가지 뚜렷한 목적으로 이중 계정 지원을 사용합니다:

  • 전략 분리: 스캘핑용 계정 (소규모, 빈번한 트레이드), 스윙 포지션용 계정 (대규모, 며칠 또는 몇 주 보유). 전략을 별도 계정에 유지하면 P&L 귀속이 명확해지고 단기 진입이 장기 포지션 관리를 방해하는 것을 방지합니다.
  • 리스크 분리: 보수적인 레버리지 한도(3~5배)의 계정 하나, 높은 확신도의 공격적인 셋업(10~20배)용 계정 하나. 공격적인 계정이 나쁜 날을 보내도, 보수적인 계정의 자산은 영향을 받지 않습니다.
  • 테스트 vs 라이브: 라이브 트레이딩용 계정 하나, 실제 자본을 투입하기 전에 새 셋업을 시도하거나 새 트레이드 유형에 익숙해지기 위한 Binance 테스트넷 계정 하나.

계정 간 전환

AIO Terminal 인터페이스 내에서 계정을 전환합니다 — 재인증 없이 두 계정의 포지션과 주문 모두 접근 가능합니다. 계정 선택기는 현재 어떤 계정에서 행동하고 있는지 표시하며, 모든 주문 탭, TP/SL 행동, 모니터링 기능이 선택된 계정의 컨텍스트에서 작동합니다.

6. 데이터 동기화: 기본 5초 간격 및 수동 새로고침

AIO Terminal은 정기적인 간격으로 Binance에서 업데이트된 포지션 및 주문 데이터를 폴링합니다. 기본 동기화 간격은 5초입니다 — 즉, 포지션 PnL, 증거금 수준, 주문 상태가 사용자 조작 없이 매 5초마다 자동으로 새로고침됩니다.

동기화 간격 조정

동기화 간격은 AIO Terminal 설정에서 구성할 수 있습니다. 낮은 간격(1~2초)은 더 많은 API 요청을 대가로 더 빠른 데이터를 제공합니다. 높은 간격(10~30초)은 API 호출 빈도를 줄이며, 종량제 연결을 사용 중이거나 주문 행동을 위한 Binance API 속도 제한 예산을 보존하고 싶을 때 유용합니다.

대부분의 활성 트레이딩 시나리오에서 기본 5초 간격은 좋은 균형을 제공합니다: API를 불필요하게 과부하시키지 않으면서, 시장 움직임 후 몇 초 내에 포지션 업데이트를 볼 수 있을 만큼 충분히 빠릅니다.

수동 동기화 옵션

마지막 5초 폴링보다 더 최신 데이터가 필요할 때 두 가지 옵션이 있습니다:

  • AIO Terminal 인터페이스의 동기화 버튼을 클릭하여 즉각적인 데이터 새로고침을 트리거합니다.
  • 브라우저 탭을 새로고침 — 이는 Binance에서 전체 세션 상태를 다시 로드하여 가장 최신 데이터를 가져옵니다. 빠르게 움직이는 일련의 트레이드 후 모든 것이 올바른 상태인지 확인하고 싶을 때 유용합니다.

어느 옵션도 활성 포지션을 방해하거나 주문을 취소하지 않습니다. Binance에서 업데이트된 데이터를 가져와 화면을 새로고침할 뿐입니다.

7. 비밀번호 관리: 최초 로그인 및 변경

AIO Terminal 접근 자격 증명은 Binance 계정 자격 증명과 다르게 작동합니다 — 이는 의도적인 설계입니다. Binance API 키(주문 배치용)와 AIO Terminal 로그인 비밀번호(터미널 접근용)는 완전히 별개입니다.

최초 로그인

AIO Terminal에 처음 접근권을 받을 때 — 일반적으로 Telegram을 통해 VIP 구독을 설정한 후 — 초기 로그인 비밀번호를 받습니다. 이 임시 비밀번호는 계정 생성 시 생성되며 최초 로그인 직후 변경해야 합니다. AIO Terminal은 최초 로그인 시 새 비밀번호를 설정하도록 안내합니다.

비밀번호 변경

비밀번호 변경은 AIO Terminal 내의 계정 설정에서 언제든지 가능합니다. 프로세스는 표준적입니다: 현재 비밀번호를 입력하고, 새 비밀번호를 입력하고, 확인합니다. Telegram 연락이나 수동 개입이 필요 없습니다.

모범 사례: AIO Terminal에 고유한 비밀번호를 사용하세요 (Binance, 이메일, 또는 다른 트레이딩 계정에서 재사용하지 마세요). AIO Terminal 자격 증명은 Binance API 키에 대한 접근을 제공합니다 — Binance 자체 인증 없이는 직접적인 자금 인출이 불가능하더라도, 손상된 비밀번호로 잃어버리는 것은 의미 있는 보안 사건입니다.

계층적 보안 모델

AIO Terminal은 두 가지 별도의 인증 계층을 사용합니다:

  1. AIO Terminal 로그인 (이메일 + 비밀번호): 대시보드 접근.
  2. Binance API 키: 터미널이 Binance 계정에서 주문을 배치하고 관리하는 권한.

어느 계층도 Binance 출금 접근권을 제공하지 않습니다 — AIO Terminal의 API 키 요구사항은 출금 권한을 명시적으로 제외합니다. 두 자격 증명 모두에 완전한 접근권을 가진 공격자도 Binance 선물 포지션과만 상호작용할 수 있을 뿐, 거래소에서 자금을 이동시킬 수 없습니다. Binance API 키 제한 추가 (AIO Terminal 서버 IP로의 IP 화이트리스트)는 이 공격 표면을 더욱 줄입니다.

종합하기: 전형적인 고급 세션

이 모든 기능을 조합하여 사용하는 트레이더의 세션이 어떻게 보이는지:

  1. AIO Terminal 열기 — 1 (키보드 단축키)을 눌러 Long 탭으로 이동. 5초 내에 Binance에서 데이터 동기화.
  2. TradingView에서 셋업 식별 — BTC가 최근 스윙 저점 아래 무효화와 함께 $65,000에서 고확률 롱 신호 표시.
  3. Long 탭에서 — Last를 눌러 $65,000을 지정가 필드에 동기화. SL에 Range 모드 선택 후 PL 클릭 — AIO Terminal이 |피벗_저점 − 진입가|를 계산하여 Range 필드를 자동으로 채움 (이 경우 $800 범위). 필요시 약간 조정.
  4. TP 설정 — PRICE 모드로 $67,500 (다음 저항 수준). 또는 PH를 눌러 최근 스윙 고점에서 TP를 채움.
  5. Long 탭 — $65,000에 지정가 주문 제출, TP $67,500, SL $64,200 ($800 Range를 통해). 세 주문 모두 동시에 실행.
  6. 6 누르기 (TP/SL 탭 키보드 단축키)으로 타겟 업데이트 — TP/SL 재설정은 기존 주문을 자동으로 취소하고 교체.
  7. ETH 롱이 계정 2에서 별도로 실행 — 동일한 워크플로우, 독립적인 포지션, 독립적인 TP/SL.
  8. BTC에 반하는 뉴스 스파이크 — 4 (청산 키보드 단축키) 누르기, BTC 포지션 구체적으로 선택, 청산. ETH 롱은 영향 없음.

이 순서의 모든 단계는 한두 번의 행동입니다. 트레이드당 총 인지적 부담은 Binance 기본 인터페이스에서의 동일한 순서보다 극적으로 낮으며, 정밀도 — 구조적으로 유효한 손절, 수수료 인식 손익분기점, 퍼센트 기반 트레일링 활성화 — 는 더 높습니다.

여전히 수동 행동이 필요한 것

AIO Terminal은 도구이지 완전한 자동화 시스템이 아닙니다. 알아야 할 몇 가지:

  • 전략 및 신호 생성: AIO Terminal은 차트를 분석하거나 진입 신호를 생성하지 않습니다. 분석에 기반하여 실행합니다. 의도된 워크플로우는: TradingView에서 분석, AIO Terminal에서 실행.
  • 레버리지 및 증거금 모드: 초기 레버리지와 교차/격리 증거금 설정은 Binance에서 관리됩니다. AIO Terminal은 저장된 레버리지를 읽고 사용하지만, 새 심볼의 초기 설정은 Binance에서 이루어집니다.
